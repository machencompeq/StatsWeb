§ 1 · 為什麼統計批改怕幻覺

LLM 本質是文字接龍，統計題正好踩中它的弱點

統計學作業包含大量數學計算（開根號、卡方分配、查 t 表）、 統計檢定判讀（p 值與 α 比較）與專有名詞。 語言模型靠機率接龍生成文字，在以下三類情境最容易「一本正經地胡說八道」：

幻覺類型 範例 後果
計算幻覺 標準差、自由度、檢定統計量算錯 把對的扣成錯、錯的給滿分
標準答案幻覺 自行腦補一套「標準解法」 與課程 Rubric 不符
判讀幻覺 p = 0.03、α = 0.05 卻說「不顯著」 結論方向相反
§ 2 · 三管齊下防幻覺

每一類幻覺，都對應一道技術防線

設計原則是「不信任模型的自由發揮」：凡是能被程式或原文釘死的環節，一律釘死。

不讓模型憑空批改

標準答案與 Rubric 以 document block 餵入並開啟引用，模型批改時必須引用 Rubric 的確切文字片段，絕不可自行假設另一套解法。

原則 → RAG + Citations

不讓模型自己算數學

檢定統計量、自由度、p 值全部由 server-side 沙箱實際執行 Python 算出， 再拿來比對學生答案；絕不用文字接龍硬算來扣分。

原則 → server-side scipy 驗算（Code Execution）

不讓模型自由發揮輸出

批改結果以強制 JSON schema 輸出（messages.parse() + Pydantic 驗證），模型聚焦在指定評分維度，大幅減少廢話與自由發揮。

原則 → Structured Outputs
注意：Citations 與 Structured Output 不可同時使用（API 會回 400）， 故 grounding 與 JSON 輸出分階段呼叫 —— 先帶引用批改與驗算，再結構化封裝。
§ 3 · 單題批改資料流

從學生按下送出，到分數落庫的五個步驟

每一份作答都走完同一條流水線；任何一站不放行，就不會有分數送到學生眼前。

  1. 1
    RAG 檢索 RAG引用
    依題號從知識庫取出該題的 Rubric 與標準答案（以題號／統計單元切塊， 確保不會撈到別的單元的答案）。
  2. 2
    批改引擎 CITATIONS CODE驗算
    Rubric 以 Citations grounding 錨定、Code Execution 寫 Python 算出正確的 t、df、p 值，adaptive thinking 逐步推理 H₀/H₁ → 統計量 → p 值 → 結論。
  3. 3
    結構化封裝 SCHEMA
    Structured Output 把批改分析封裝為固定 JSON：逐維度得分、引用片段、 驗算數值，一欄都不能多、不能少。
  4. 4
    教授視角覆核（雙模型） 教授覆核
    第二階段由「統計學教授」視角檢查：有沒有把對的判錯、錯的判對？ 是否與程式計算結果一致？由 claude-opus-4-8 與 claude-sonnet-4-6 雙模型獨立覆核，回傳結論與信心度。
  5. 5
    信心分流
    信心度足夠 → 結果落庫 ／ 不足 → 先交雙模型仲裁（claude-sonnet-4-6 獨立重批， 兩模型一致即落庫、免人工）／ 仲裁仍分歧 → 進人工複核佇列
§ 4 · SPDC 作答框架

四段式作答，四個維度給分

學生依 State → Plan → Do → Conclude 四段作答，AI 也照同樣四個維度逐項評分， 每維度 4 分、總分 16 分 —— 作答結構即評分結構。

S

State 陳述假設

寫出 H₀ 與 H₁，並定義母體參數（如 μ 代表什麼）。

4 pts
P

Plan 選擇方法與條件

選擇檢定方法，並檢查條件：隨機性、常態性／CLT、獨立性。

4 pts
D

Do 執行計算

算出檢定統計量、自由度與 p 值 —— 這一段由 scipy 逐項驗算。

4 pts
C

Conclude 脈絡化結論

依 p 值與 α 下結論，並回扣題目脈絡（不能只寫「拒絕 H₀」）。

4 pts
§ 5 · 數學符號輸入層

人看得懂 ↔ 電腦看得懂

學生看到的是課本上熟悉的符號（x̄、σ²、H₀…），電腦保存的是標準編碼 LaTeX：學生按下符號鈕，系統插入對應編碼，畫面以 KaTeX 即時渲染回符號，送出時保存的是機器可讀、可直接送批改的原始碼 —— 學生完全不需要懂 LaTeX。

迷你體驗：點一顆符號鍵，看電腦存了什麼

電腦看到的 （點上方任一符號鍵）

正式版以 MathLive 虛擬鍵盤輸出 LaTeX、KaTeX 渲染回符號；本頁僅示範「符號 ↔ 編碼」的對應概念。

§ 6 · 常見問題

學生最常問的四件事

AI 會不會算錯數學？
不會用「文字接龍」的方式算。所有檢定統計量、自由度、p 值都由 Code Execution 在沙箱裡以 scipy 實際執行算出，AI 只負責拿程式輸出去比對你的答案 —— 數值判定一律以程式輸出為依據。
AI 憑什麼扣我分？
每一個扣分都必須引用 Rubric 原文的確切片段（Citations 錨定）， 你在批改結果裡看得到「這 1 分是依據評分標準哪一句扣的」。 AI 被明確禁止自行發揮另一套標準解法。
AI 沒把握（低信心）的時候怎麼辦？
批改結果會經第二階段「教授視角」覆核並給出信心度；低於門檻或被標記「需修正」的案件， 自動進入人工複核佇列，由授課教師親自裁定，不會讓沒把握的分數直接送到你手上。
我的作答會被存成什麼格式？
你按符號鈕輸入的數學式會轉成 LaTeX 保存 —— 一種機器可讀的標準編碼， 能精準還原上下標、分數與希臘字母，之後可直接餵入批改引擎，也能隨時渲染回你熟悉的符號。
